#+TITLE: Emacs overlay for Nixpkgs
#+AUTHOR: Desmond Wang

* Emacs overlay for Nixpkgs

This is a small Emacs overlay for Nixpkgs.

It provides custom Emacs builds and a lightweight helper,
~emacsWithPackagesFromSetup~, which scans Emacs Lisp configuration files for
literal ~setup.el~ forms and builds an Emacs closure containing the packages
declared with ~:package~.

This fork intentionally removes the older helpers:

- ~emacsWithPackagesFromUsePackage~
- ~emacsWithPackagesFromPackageRequires~

* Contents of the overlay

This overlay consists of two main parts:

- ~emacs~ overlay
- ~package~ overlay

You can use the whole overlay, or import only the part you need.

** ~package~ overlay

The package overlay updates Emacs package sets, including package metadata from
ELPA, MELPA, MELPA stable, and NonGNU ELPA.

This is useful when you want newer Emacs packages than the ones available in
your current Nixpkgs revision.

** ~emacs~ overlay

The Emacs overlay provides custom Emacs builds.

Common attributes include:

#+begin_src nix
pkgs.emacs-git
pkgs.emacs-git-pgtk

pkgs.emacs-igc
pkgs.emacs-igc-pgtk
#+end_src

* Quickstart

#+begin_src nix
{
  description = "My Emacs setup";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

    emacs-overlay = {
      url = "github:dezzw/emacs-overlay";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { self, nixpkgs, emacs-overlay }:
    let
      system = "x86_64-linux";
      pkgs = import nixpkgs {
        inherit system;
        overlays = [
          emacs-overlay.overlays.default
        ];
      };
    in
    {
      packages.${system}.default = pkgs.emacs-git;
    };
}
#+end_src

* Building Emacs with packages from setup.el

This fork provides:

#+begin_src nix
pkgs.emacsWithPackagesFromSetup
#+end_src

It scans one or more Emacs Lisp files for literal ~setup~ forms and extracts
packages declared with ~:package~.

Example ~init.el~:

#+begin_src emacs-lisp
(require 'setup)

(setup magit
  (:package magit))

(setup vertico
  (:package vertico))

(setup orderless
  (:package orderless))

(setup corfu
  (:package corfu))
#+end_src

Then in Nix:

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;
  config = ./init.el;
}
#+end_src

* Accepted config values

~emacsWithPackagesFromSetup~ accepts a single file, a list of files, a directory,
or a mixed list of files and directories.

** Single file

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;
  config = ./init.el;
}
#+end_src

** Multiple files

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;

  config = [
    ./early-init.el
    ./init.el
    ./modules/completion.el
    ./modules/git.el
  ];
}
#+end_src

** Directory

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;
  config = ./modules;
}
#+end_src

All ~.el~ files inside the directory are scanned recursively.

** Mixed files and directories

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;

  config = [
    ./init.el
    ./modules
    ./site-lisp
  ];
}
#+end_src

** Raw Emacs Lisp string

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;

  config = ''
    (setup magit
      (:package magit))

    (setup vertico
      (:package vertico))
  '';
}
#+end_src

* Using extra packages

Some packages or tools may not be declared in your Emacs config.

For example, language servers, command line tools, formatters, or packages loaded
dynamically may need to be added manually.

Use ~extraEmacsPackages~:

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;
  config = ./init.el;

  extraEmacsPackages = epkgs: [
    epkgs.vterm
    epkgs.treesit-grammars.with-all-grammars

    pkgs.ripgrep
    pkgs.fd
    pkgs.shellcheck
  ];
}
#+end_src

* Overriding Emacs packages

You can override packages using the ~override~ argument.

Example:

#+begin_src nix
pkgs.emacsWithPackagesFromSetup {
  package = pkgs.emacs-git-pgtk;
  config = ./init.el;

  override = final: prev: {
    magit = prev.magit.overrideAttrs (old: {
      patches = (old.patches or [ ]) ++ [
        ./patches/magit.patch
      ];
    });
  };
}
#+end_src

* Troubleshooting

** Package not found

If you see an error like:

#+begin_example
Emacs package 'foo' declared by setup.el :package was not found.
#+end_example

Then either:

- the package name does not match the Nix Emacs package attribute;
- the package is not available in your package set;
- the package must be added manually through ~extraEmacsPackages~;
- or your package overlay is not enabled.

Example manual addition:

#+begin_src nix
extraEmacsPackages = epkgs: [
  epkgs.foo
];
#+end_src

** Custom setup macros are ignored

The parser does not macroexpand your config.

This works:

#+begin_src emacs-lisp
(setup magit
  (:package magit))
#+end_src

This does not automatically work:

#+begin_src emacs-lisp
(my/setup magit)
#+end_src

Use literal ~setup~ forms or add packages manually through
~extraEmacsPackages~.

** Directory config scans too much

Only ~.el~ files are scanned. If your directory contains generated files,
temporary files, or archived configs, move them outside the scanned directory or
pass a more precise list of files.

For example:

#+begin_src nix
config = [
  ./init.el
  ./modules/completion.el
  ./modules/git.el
  ./modules/ui.el
];
#+end_src

* Design goals

This fork tries to stay small and explicit:

- keep the Emacs build overlay;
- keep the updated Emacs package metadata;
- remove old use-package and Package-Requires helpers;
- use setup.el as the only automatic config scanner;
- support single files, multiple files, and directories;
- keep manual escape hatches through ~extraEmacsPackages~ and ~override~.
